CONCEITOS BÁSICOS

Quando a Microsoft iniciou a criação do .NET Framework 3.0, um entre quatro dos grandes pilares que havia dentro dele era o Indigo que mais tarde recebeu o nome de Windows Communication Foundation, ou simplesmente WCF. O WCF unificou as várias tecnologias de programação distribuídas na plataforma Microsoft em um único modelo, baseando-se na arquitetura orientada à serviços (SOA). Facilitou consideravelmente o aprendizado e desenvolvimento, já que o WCF está totalmente desacoplado das regras de negócios que serão expostas pelo serviço.

CONCEITOS BÁSICOS

  • • Seu principal objetivo é permitir que analistas e desenvolvedores criem aplicações voltadas para computação distribuída.
  • • Possui ainda um conjunto de bibliotecas (classes) que permitem aos desenvolvedores criar estas aplicações para funcionarem sob o sistema operacional Windows.
  • • Você não tem que aprender as muitas tecnologias que estão envolvidas na construção de um aplicativo distribuído. Quando você usa um Framework como o WCF, o runtime do WCF trata estas questões automaticamente.

PRINCIPAIS VANTAGENS

  • • Fornece maior confiabilidade e segurança em relação às tecnologias anteriores.
  • • O WCF é interoperável com outros serviços, quando comparado ao Remoting, onde o Desktop e o Cliente necessitam ser .NET. Esta vantagem abre portas para a construção de aplicações paralelas ao ERP em outras tecnologias, agregando assim mais valor à ferramenta.
  • • Melhora o tráfego de informações pela rede, a fim de diminuir informações desnecessárias.
  • • Flexibiliza e aumenta as possibilidades de interação dos serviços com outros sistemas.

PRINCIPAIS VANTAGENS

  • • Qualquer aplicação pode ser servidor para o WCF, isto é, não preciso de um servidor Web para nenhum tipo de tarefa.
  • • Posso contar com recursos do Windows para ativar o serviço com a chegada de uma nova mensagem.
  • • Temos grande liberdade para determinar como será a passagem de mensagens de um lado para o outro.
  • • Flexibilidade para configurar o serviço, muitas vezes, evitando que qualquer linha de código seja escrita.
  • • Não é necessário aprender Remoting, COM+, Enterprise Services, etc… – basta aprender o conceito de Web Services e o WCF em si.

O WCF fornece uma grande quantidade de funcionalidades que facilmente podem ser adicionadas em serviços. Além disso, grande parte dessas funcionalidades podem ser configuradas de forma declarativa, através de arquivos de configuração que, na maioria dos casos, traz uma enorme flexibilidade. O uso de WCF ainda está sendo descoberto pelas empresas não só do Brasil, mas do mundo. Há muita coisa pronta feita em Web Services “normais” do .NET e inclusive, novos projetos ainda hoje são criados usando esta tecnologia considerada legada pela Microsoft, mas não ainda pelo mercado. O grande argumento para a não migração é a falta de pessoal capacitado para utilizar esta nova tecnologia.

ESTRUTURA

A estrutura de um serviço WCF não é muito complexa, pois devemos utilizar conceitos puros de programação .NET para a criação do contrato e da classe que representará o serviço. Além disso, o WCF também suporta a utilização de tipos complexos, como classes que criamos para atender uma determinada necessidade.

Para que possamos criar, projetar e implantar serviços WCF, devemos nos ater a alguns conceitos básicos que são os EndPoints e seus componentes ( Address, Binding e Contract).

REST

Representational State Transfer

(Transferência de Estado Representacional)

SURGIMENTO DO REST

  • • Década de 1990: surge a necessidade de se realizar transações pela Internet.
  • • Novo paradigma: como evoluir de uma web de páginas para uma web de serviços?
  • • Dr. Roy Fielding, (co-criador do protocolo HTTP), sugere o termo REST em sua tese de doutorado no ano 2000.

Roy Fielding

INTRODUÇÃO AO REST

  • • É uma técnica de engenharia de software para sistemas hipermídia distribuídos como o World Wide Web.
  • • REST: considerado um estilo arquitetural e não uma especificação.

PRINCÍPIOS BÁSICOS DO REST

  • • Protocolo cliente/servidor sem estado: nem o cliente e nem o servidor necessitam gravar nenhum estado das comunicações.
  • • Sintaxe universal para identificar os recursos, que são direcionados pela URI.
  • • Métodos padronizados: conjunto de operações bem definidos que se aplicam a todos os recursos de informação.

FUNCIONAMENTO DO REST

  • • Baseado no mesmo funcionamento da web de páginas.
  • • Serviços devem ser simples e abertos e qualquer pessoa, em qualquer lugar, utilizando qualquer plataforma deve ser capaz de utilizá-los.
  • • Devem ser baseados em um protocolo universal (Para a web pode ser o HTTP).

FUNCIONAMENTO DO REST

  • • Os serviços que seguem este estilo arquitetural são conhecidos como serviços RESTful e se baseiam no funcionamento da web e do protocolo HTTP.
  • • A exposição do endereço do serviço na rede é feita através de uma URI.
  • • Os serviços devem também expor recursos que possam ser associados através de URIs.

FUNCIONAMENTO DO REST

  • • As chamadas às URIs são realizadas através de verbos HTTP.
  • • O verbo utilizado na requisição indica para o serviço que a ação deve ser realizada com o recurso.
  • • Os verbos mais comuns para serem utilizados com o REST são GET, POST, PUT e DELETE.

XML vs. JSON

XML

Extensible Markup Language

É uma recomendação da W3C para gerar linguagens de marcação para necessidades especiais.

Seu propósito principal é a facilidade de compartilhamento de informações tanto na Internet como no mundo corporativo.

XML

CARACTERÍSTICAS

  • • Estrutura mais complexa
  • • Elementos precisam ter nomes
  • • Elementos precisam de chaves de inicio e fim
  • • Por ter estrutura mais complexa, pode não ser adequado para grande transferência de dados
  • • Funciona muito bem para descrição de dados
  • • Possibilidade de adicionar filhos e atributos a cada elemento

JSON

JavaScript Object Notation

É um formato leve para intercâmbio de dados computacionais.

JSON é um subconjunto da notação de objeto de JavaScript, mas seu uso não requer Javascript exclusivamente.

JSON

CARACTERÍSTICAS E VANTAGENS

  • • É simples e mais rápido.
  • • Alternativa para requisições AJAX.
  • • É praticamente um array.
  • • Adequado para transferência de dados na rede.
  • • A maioria das linguagens server-side oferece suporte.
